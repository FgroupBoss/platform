## 二、需求阶段 AI 能力

### 2.1 AI 需求扩写与润色（P0 高价值）

**触发位置**：需求创建/编辑页

**场景**：产品经理输入一句话需求，AI 自动扩写为结构化需求文档。

**输入**：
```
用户输入: "做一个用户登录功能"
```

**AI 输出**：
```markdown
## 需求标题
用户登录功能

## 需求描述
实现用户通过邮箱/手机号 + 密码方式登录系统，支持记住密码和自动登录。

## 功能点
1. 邮箱 + 密码登录
2. 手机号 + 验证码登录
3. 密码错误 3 次锁定账户 30 分钟
4. 支持"记住我"功能（7 天免登录）
5. 登录成功后跳转到用户上次访问页面

## 验收标准
- [ ] 正确邮箱+密码可成功登录，跳转首页
- [ ] 错误密码提示"用户名或密码错误"，不泄露具体原因
- [ ] 连续 3 次错误后锁定，提示剩余锁定时间
- [ ] 手机验证码 60 秒内有效，过期需重新获取
- [ ] 勾选"记住我"后关闭浏览器再打开仍保持登录

## 影响范围
- 前端：登录页面、注册页面、密码找回页面
- 后端：用户认证服务、验证码服务
- 数据库：用户表、登录日志表
```

**实现方式**：
- 调用 LLM，Prompt 模板注入公司需求规范
- 用户可对生成结果进行修改后确认保存
- 支持"重新生成"和"回退"操作

---

### 2.2 AI 需求评审助手（P0 高价值）

**触发位置**：需求提交评审时 / 需求详情页新增"AI 评审"按钮

**场景**：AI 自动检查需求质量，给出评审意见，减少人工评审遗漏。

**检查维度**：

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 完整性检查 | 是否缺少验收标准、影响范围 | 高 |
| 清晰度检查 | 描述是否模糊、有歧义 | 中 |
| 边界条件 | 是否遗漏异常场景、边界值 | 高 |
| 依赖检查 | 是否标注了前后端依赖 | 中 |
| 安全考量 | 是否涉及敏感数据、权限控制 | 高 |
| 重复检测 | 系统中是否已有相似需求 | 中 |

**AI 输出示例**：
```markdown
## AI 评审报告

### 严重问题 (2)
- [ ] 缺少异常场景描述：未说明网络超时、服务不可用时的处理方式
- [ ] 未标注是否需要接入验证码防刷机制

### 建议优化 (3)
- [ ] "记住我"功能未明确 Cookie 有效期策略
- [ ] 建议补充登录失败日志记录的存储周期
- [ ] 验收标准建议补充"并发登录"场景

### 相似需求提醒
- REQ-015「第三方登录集成」(相似度 72%) — 建议确认是否合并
```

---

### 2.3 AI 需求优先级智能推荐（P1 高价值）

**触发位置**：需求列表 / 需求创建时

**场景**：根据需求属性自动推荐优先级，减少人工排序争议。

**AI 输入因子**：
- 是否阻塞其他需求
- 涉及用户数 / 影响面
- 是否为合规/安全需求
- 预估开发工时
- 业务方期望上线时间

**输出**：推荐 P0/P1/P2/P3 + 推荐理由。

---

### 2.4 AI 需求相似度检测（P1 选做）

**触发位置**：需求创建时实时检测

**场景**：输入需求标题和描述后，自动在已有需求库中检索相似需求，避免重复建需求。

**实现**：RAG（检索增强生成）方案，用 Embedding 向量检索 + LLM 判断相似度。

### 3.2 AI 工时预估（P2 选做）

**触发位置**：任务创建时

**场景**：AI 根据任务描述和历史数据估算工时，辅助 Sprint 容量规划。

**输入**：任务标题 + 描述 + 类型 + 历史相似任务工时数据

**输出**：预估工时（小时）+ 置信度区间。

### 3.3 AI 代码审查（P1 选做，需 Git 集成）

**触发位置**：MR/PR 关联到任务时

**场景**：AI 自动审查代码变更，输出审查意见，标注潜在问题。

**检查维度**：安全漏洞、代码规范、性能问题、逻辑错误。

---

### 3.4 AI 提交信息生成（P2 探索）

**触发位置**：IDE 插件 / Git Hook

**场景**：AI 根据代码 diff 自动生成规范的 Commit Message。

### 4.2 AI 测试用例评审（P1 选做）

**触发位置**：用例创建/编辑时

**场景**：AI 检查用例质量，给出优化建议。

**检查项**：
- 步骤是否清晰可执行
- 期望结果是否明确可验证
- 是否覆盖边界条件

### 4.3 缺陷相关 AI 能力

#### 4.3.1 AI 缺陷智能分类（P0 高价值）

**触发位置**：缺陷创建时

**场景**：用户输入缺陷描述后，AI 自动推荐：严重程度、关联模块、建议指派修复人。

**输入**：缺陷标题 + 描述 + 复现步骤

**AI 输出**：
```json
{
  "severity_suggestion": "major",
  "severity_reason": "验证码功能完全不可用，阻塞核心登录流程",
  "module": "用户认证",
  "suggested_assignee": "张三",
  "assignee_reason": "张三最近修改过验证码相关代码，且修复过相似缺陷 BUG-003",
  "related_defects": ["BUG-003", "BUG-012"]
}
```

---

#### 4.3.2 AI 缺陷去重（P0 高价值）

**触发位置**：缺陷创建时实时检测

**场景**：提交缺陷时，AI 自动检索是否已有相似缺陷，避免重复提交。

**实现**：RAG 检索 + LLM 相似度判断，相似度 > 80% 时提示用户。

**提示示例**：
```
⚠️ 检测到相似缺陷

BUG-003「验证码发送失败」(相似度 89%)
状态：修复中 | 修复人：张三
创建时间：2026-06-02

建议：是否仍要提交？如确认不同，请补充差异说明。
[仍要提交] [取消，查看 BUG-003]
```

---

#### 4.3.3 AI 根因分析（P1 选做）

**触发位置**：缺陷详情页

**场景**：AI 结合缺陷描述、关联代码变更、相似缺陷历史，推测可能的根因。

**AI 输出**：
```markdown
## AI 根因分析

### 推测根因
验证码服务调用超时，可能是短信服务商接口响应变慢导致。

### 分析依据
- 关联任务 T-003「注册接口」最近变更了验证码调用方式
- 历史缺陷 BUG-003 也是验证码相关问题，原因为超时时间设置过短
- 当前 timeout 设置: 3000ms，建议调整为 5000ms

### 建议排查方向
1. 检查短信服务商近期 SLA
2. 检查网络链路延迟
3. 考虑增加重试+降级机制
```

---

#### 4.3.4 AI 修复建议（P2 探索）

**触发位置**：缺陷详情页

**场景**：AI 根据缺陷描述和关联代码，给出修复建议甚至代码片段。

## 五、运营与管理 AI 能力

### 5.1 AI 智能周报/日报生成（P0 高价值）

**触发位置**：项目仪表盘 / 个人工作台

**场景**：AI 自动汇总本周/本日项目动态，生成结构化报告。

**AI 输入**：本周创建/完成的需求、任务、缺陷、操作日志、评论

**AI 输出**：
```markdown
## Sprint 3 周报 (06/01 - 06/07)

### 整体进度
- 需求完成率：60%（3/5），1 个延期风险
- 缺陷：新增 8 个，修复 5 个，剩余 3 个
- 测试通过率：75%

### 本周亮点
- ✅ 用户登录功能完成开发和测试，已发布
- ✅ 首页数据看板完成联调，进入测试阶段

### 风险项
- ⚠️ REQ-005「消息通知模块」开发进度滞后 2 天，建议协调资源
- ⚠️ BUG-001「验证码发送失败」阻塞登录测试，需优先修复

### 下周计划
- 完成消息通知模块开发
- 启动 Sprint 3 回归测试
- 修复剩余 3 个缺陷
```

---

### 5.2 AI 智能问答（P0 高价值）

**触发位置**：全局底部 AI 助手入口 / 自然语言搜索框

**场景**：用自然语言查询项目状态，AI 理解意图并返回数据。

**用户提问示例**：
- "张三这周完成了哪些任务？"
- "Sprint 3 还差多少没做完？"
- "最近一周新增了多少缺陷？"
- "哪些需求还没写测试用例？"
- "哪个需求缺陷最多？"

**AI 处理流程**：
```
用户自然语言 → LLM 意图识别 → 转换为数据库查询 → 返回格式化结果 + 自然语言解释
```

**实现方式**：Text-to-SQL + RAG 结合。

---

### 5.3 AI 项目风险预测（P1 选做）

**触发位置**：项目仪表盘

**场景**：AI 分析历史数据，预测迭代延期风险、缺陷爆发趋势。

**AI 输入**：
- 当前迭代进度 vs 历史同期进度
- 缺陷新增趋势
- 任务完成速率
- 人员负载

**AI 输出**：
```markdown
## AI 风险预警

### Sprint 3 延期风险：高 (78%)
- 当前完成率 60%，历史同期均值 72%
- 剩余 8 天，按当前速度预计完成 85%
- 风险点：REQ-005 开发进度滞后

### 缺陷爆发预警：中
- 本周新增缺陷 8 个，上周 3 个，增幅 167%
- 建议暂停新功能开发，优先修复已有缺陷

### 人员负载预警
- 张三：本周已登记 32h，超出负载(30h)
```

### 5.4 AI 需求/缺陷/会议智能摘要（P1 选做）

**触发位置**：需求详情页 / 缺陷详情页 / 操作日志

**场景**：对长文本、大量评论、操作日志进行 AI 摘要，快速了解上下文。

**示例**：

```
该需求「用户登录」经 3 轮讨论，核心争议点在于：

3. 验证码方案从自建改为对接第三方服务

当前状态：开发中，预计 6 月 5 日提测。
```

## 八、关键设计原则

1. **AI 辅助而非替代**：所有 AI 输出都是"建议"，最终决策权在用户，不可自动生效
2. **渐进式采纳**：AI 功能入口为按钮触发，不打断用户原有流程
3. **可解释性**：AI 输出附带理由，让用户理解为什么这么建议
4. **成本可控**：简单任务用轻量模型，复杂任务用强模型；高频调用加缓存
5. **数据安全**：敏感数据不出域，Prompt 中不传密码/密钥等敏感信息
6. **反馈闭环**：每个 AI 输出支持 👍/👎 反馈，持续优化效果
