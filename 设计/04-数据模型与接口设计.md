## 二、数据表详细设计

### 2.1 users（用户表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 用户 ID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| email | VARCHAR(100) | UNIQUE, NOT NULL | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 加密密码 |
| display_name | VARCHAR(100) | NOT NULL | 显示名称 |
| avatar_url | VARCHAR(500) | NULL | 头像 URL |
| role | ENUM('admin','project_admin','pm','dev_lead','dev','qa_lead','qa','viewer') | DEFAULT 'dev' | 全局角色 |
| status | ENUM('active','disabled') | DEFAULT 'active' | 账号状态 |
| created_at | DATETIME | DEFAULT NOW() | |
| updated_at | DATETIME | ON UPDATE NOW() | |

### 2.2 projects（项目表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | |
| name | VARCHAR(200) | NOT NULL | 项目名称 |
| key | VARCHAR(10) | UNIQUE, NOT NULL | 项目标识(如 DEMO) |
| description | TEXT | NULL | 项目描述 |
| status | ENUM('active','archived') | DEFAULT 'active' | |
| created_by | BIGINT | FK → users.id | |
| created_at | DATETIME | DEFAULT NOW() | |
| updated_at | DATETIME | ON UPDATE NOW() | |

### 2.3 project_members（项目成员表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | |
| project_id | BIGINT | FK → projects.id, NOT NULL | |
| user_id | BIGINT | FK → users.id, NOT NULL | |
| role | ENUM('project_admin','pm','dev_lead','dev','qa_lead','qa','viewer') | NOT NULL | 项目内角色 |
| joined_at | DATETIME | DEFAULT NOW() | |
| **UNIQUE(project_id, user_id)** | | | |

### 2.4 sprints（迭代表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | |
| project_id | BIGINT | FK → projects.id, NOT NULL | |
| name | VARCHAR(200) | NOT NULL | 迭代名称 |
| goal | TEXT | NULL | 迭代目标 |
| start_date | DATE | NOT NULL | |
| end_date | DATE | NOT NULL | |
| status | ENUM('planning','active','completed') | DEFAULT 'planning' | |
| created_at | DATETIME | DEFAULT NOW() | |
| updated_at | DATETIME | ON UPDATE NOW() | |

### 2.5 requirements（需求表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | |
| project_id | BIGINT | FK → projects.id, NOT NULL | |
| sprint_id | BIGINT | FK → sprints.id, NULL | 所属迭代 |
| title | VARCHAR(500) | NOT NULL | 需求标题 |
| description | TEXT | NULL | 需求描述 |
| acceptance_criteria | TEXT | NULL | 验收标准 |
| priority | ENUM('P0','P1','P2','P3') | DEFAULT 'P2' | |
| status | ENUM('draft','reviewed','in_dev','in_test','released','closed') | DEFAULT 'draft' | |
| owner_id | BIGINT | FK → users.id, NULL | 产品负责人 |
| created_by | BIGINT | FK → users.id, NOT NULL | |
| created_at | DATETIME | DEFAULT NOW() | |
| updated_at | DATETIME | ON UPDATE NOW() | |

**索引**：
- INDEX `idx_project_status` (project_id, status)
- INDEX `idx_sprint` (sprint_id)

### 2.6 dev_tasks（研发任务表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | |
| requirement_id | BIGINT | FK → requirements.id, NOT NULL | |
| title | VARCHAR(500) | NOT NULL | |
| description | TEXT | NULL | |
| assignee_id | BIGINT | FK → users.id, NULL | |
| status | ENUM('todo','in_progress','done','tested') | DEFAULT 'todo' | |
| type | ENUM('frontend','backend','fullstack','data','ops','other') | DEFAULT 'other' | |
| priority | ENUM('P0','P1','P2','P3') | DEFAULT 'P2' | |
| branch_name | VARCHAR(200) | NULL | 关联 Git 分支 |
| estimated_hours | DECIMAL(5,1) | NULL | 预估工时 |
| actual_hours | DECIMAL(5,1) | NULL | 实际工时 |
| created_by | BIGINT | FK → users.id, NOT NULL | |
| created_at | DATETIME | DEFAULT NOW() | |
| updated_at | DATETIME | ON UPDATE NOW() | |

**索引**：
- INDEX `idx_requirement` (requirement_id)
- INDEX `idx_assignee_status` (assignee_id, status)

### 2.7 test_cases（测试用例表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | |
| requirement_id | BIGINT | FK → requirements.id, NOT NULL | |
| title | VARCHAR(500) | NOT NULL | |
| type | ENUM('functional','api','performance','security','other') | DEFAULT 'functional' | |
| priority | ENUM('P0','P1','P2','P3') | DEFAULT 'P2' | |
| preconditions | TEXT | NULL | 前置条件 |
| steps | JSON | NOT NULL | 测试步骤 |
| expected_result | TEXT | NOT NULL | 期望结果 |
| last_result | ENUM('pass','fail','blocked','pending') | DEFAULT 'pending' | 最近执行结果 |
| automation | BOOLEAN | DEFAULT FALSE | 是否自动化 |
| created_by | BIGINT | FK → users.id, NOT NULL | |
| created_at | DATETIME | DEFAULT NOW() | |
| updated_at | DATETIME | ON UPDATE NOW() | |

**steps JSON 格式**：
```json
[
  { "step": 1, "action": "打开登录页面", "expected": "显示登录表单" },
  { "step": 2, "action": "输入邮箱 test@example.com", "expected": "邮箱格式验证通过" },
  { "step": 3, "action": "输入密码 123456", "expected": "密码输入框显示掩码" },
  { "step": 4, "action": "点击登录按钮", "expected": "跳转到首页" }
]
```

## 三、API 接口设计

### 3.1 通用规范

| 项目 | 规范 |
|------|------|
| Base URL | `/api/v1` |
| 认证方式 | Header `Authorization: Bearer <jwt_token>` |
| 请求格式 | `application/json` |
| 响应格式 | `{ "code": 0, "message": "success", "data": {} }` |
| 分页参数 | `?page=1&page_size=20` |
| 分页响应 | `{ "data": { "list": [...], "total": 100, "page": 1, "page_size": 20 } }` |

### 3.2 错误码定义

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1001 | 参数错误 |
| 1002 | 资源不存在 |
| 1003 | 无权限 |
| 1004 | 未登录/Token 过期 |
| 1005 | 状态流转不合法 |
| 2001 | 用户名已存在 |
| 2002 | 邮箱已存在 |
| 2003 | 密码错误 |
| 5000 | 服务器内部错误 |
